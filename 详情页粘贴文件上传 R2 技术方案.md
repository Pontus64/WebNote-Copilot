# 详情页粘贴文件上传 R2 技术方案

## 1. 背景与目标

当前笔记详情页已经切换为 Milkdown Markdown 编辑器，但粘贴图片时仍是占位 TODO。下一步要实现接近 Typora 的粘贴体验：用户在详情页粘贴图片、视频或其他文件后，内容应立即出现在光标位置，不弹确认框，不要求用户先手动保存。

目标效果：

- 粘贴图片后立即显示图片预览，上传完成后把临时地址替换为 R2 公开 URL。
- 粘贴视频后立即显示视频占位或预览，上传完成后生成可播放的 `<video controls>`。
- 粘贴普通文件后立即显示附件链接，上传完成后替换为 R2 公开 URL。
- 二进制文件存 R2，附件元数据和笔记关联存 D1。
- 主站入口和嵌入脚本入口行为一致。
- R2 上传能力异常时，只影响附件上传，不影响笔记列表、详情编辑、普通文本保存。

## 2. 技术选型与边界

### 2.1 存储分层

- R2：保存图片、视频、音频、PDF、zip、文本附件等二进制对象。
- D1：保存附件元数据，包括 `note_id`、`user_id`、`r2_key`、`public_url`、文件名、MIME、大小、类型、创建时间等。
- Notes 表：继续只保存 Markdown 正文，不保存文件 blob。

这能避免 D1 行大小和查询限制被大文件拖垮，也符合当前 Cloudflare Worker + D1 + R2 架构。

### 2.2 公开访问策略

采用 R2 自定义公开域名作为正式 Markdown URL，例如：

```text
https://assets.notes.edmund.xin
```

实施时通过环境变量配置：

```text
NOTE_ASSETS_PUBLIC_BASE_URL=https://assets.notes.edmund.xin
```

`r2.dev` 仅作为本地或测试 fallback，不写入生产笔记 Markdown。Cloudflare 文档明确 `r2.dev` 是开发用途，正式长期链接应使用自定义域名。

### 2.3 Typora 式行为边界

用户粘贴文件时，不出现“是否保存笔记”提示，也不要求用户手动点击保存。

对未保存的新笔记：

1. 编辑器先在光标处插入本地 `blob:` 预览或附件占位。
2. 前端后台静默创建笔记，获取 `noteId`。
3. 上传文件到 `/api/notes/:noteId/assets`。
4. 上传完成后替换临时 Markdown 为公开 R2 URL。
5. 上传失败时保留失败占位并显示 toast。

注意：这个“后台先保存”只是实现细节，不能表现成用户需要先保存。

## 3. 数据模型

当前 `migrations/0004_rebuild_notes_v2.sql` 已经有 `note_assets` 基础表。新增迁移补齐字段，建议目标结构如下：

```sql
CREATE TABLE note_assets (
	id TEXT PRIMARY KEY,
	note_id TEXT NOT NULL,
	user_id TEXT NOT NULL,
	r2_key TEXT NOT NULL UNIQUE,
	public_url TEXT NOT NULL,
	file_name TEXT NOT NULL,
	mime_type TEXT NOT NULL,
	byte_size INTEGER NOT NULL,
	asset_kind TEXT NOT NULL,
	created_at INTEGER NOT NULL,
	updated_at INTEGER NOT NULL,
	deleted_at INTEGER,
	FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
	FOREIGN KEY (user_id) REFERENCES auth_users(id)
);

CREATE INDEX idx_note_assets_note_active
ON note_assets(note_id, deleted_at, created_at ASC);

CREATE INDEX idx_note_assets_user_note
ON note_assets(user_id, note_id);
```

`asset_kind` 取值：

- `image`
- `video`
- `audio`
- `document`
- `archive`
- `file`

R2 key 规则：

```text
users/{userId}/notes/{noteId}/{assetId}-{safeFileName}
```

`safeFileName` 只保留安全字符，原始文件名仍存 `file_name` 用于展示。

## 4. Worker API 设计

### 4.1 上传附件

```http
POST /api/notes/:noteId/assets
Authorization: Bearer <sessionToken>
Content-Type: image/png
X-File-Name: screenshot.png
X-File-Size: 123456
```

请求体直接传单个文件二进制。使用 raw body 可以减少 multipart 解析复杂度，避免在 Worker 里额外复制大对象。

返回：

```json
{
	"id": "asset-id",
	"noteId": "note-id",
	"fileName": "screenshot.png",
	"mimeType": "image/png",
	"byteSize": 123456,
	"assetKind": "image",
	"publicUrl": "https://assets.notes.edmund.xin/users/.../asset-id-screenshot.png",
	"markdown": "![screenshot.png](https://assets.notes.edmund.xin/users/.../asset-id-screenshot.png)",
	"createdAt": 1770000000000
}
```

处理流程：

1. `requireAuth` 校验登录。
2. 校验 note 是否属于当前用户。
3. 校验 MIME、文件大小、文件名。
4. 生成 `assetId` 和 `r2_key`。
5. `env.NOTE_ASSETS.put(r2Key, request.body, { httpMetadata })` 写入 R2。
6. 写入 D1 `note_assets`。
7. 返回公开 URL 和可插入的 Markdown。

### 4.2 附件列表

```http
GET /api/notes/:noteId/assets
```

返回当前笔记未删除附件列表，用于后续做附件管理面板。

### 4.3 删除附件

```http
DELETE /api/notes/:noteId/assets/:assetId
```

处理流程：

1. 校验当前用户拥有该笔记和附件。
2. D1 设置 `deleted_at`。
3. 删除 R2 对象。

### 4.4 删除笔记时清理附件

现有 `deleteNote` 需要扩展：

1. 查询当前笔记所有未删除附件的 `r2_key`。
2. 删除 notes 记录。
3. 批量删除 R2 对象。

D1 外键 cascade 只处理元数据，不能自动删除 R2 对象，所以 Worker 必须显式清理。

## 5. 前端设计

### 5.1 API client

在 `client/shared/notesApi.ts` 增加：

```ts
export type NoteAsset = {
	id: string;
	noteId: string;
	fileName: string;
	mimeType: string;
	byteSize: number;
	assetKind: "image" | "video" | "audio" | "document" | "archive" | "file";
	publicUrl: string;
	markdown: string;
	createdAt: number;
};

export function uploadNoteAsset(
	noteId: string,
	file: File,
	apiBase = ""
): Promise<NoteAsset>;
```

上传时不要手动设置 JSON `Content-Type`，而是设置为文件本身 MIME。

### 5.2 MarkdownNoteEditor 改造

当前 `MarkdownNoteEditor` 已经有粘贴图片 TODO。改造方向：

- 不再只识别图片，而是读取 `event.clipboardData.files`。
- 非文件粘贴保持 Milkdown 默认行为。
- 文件粘贴时阻止默认行为，调用 `onPasteFiles(files)`。
- 对外暴露 `insertMarkdown(markdown)` 或 `replaceMarkdownToken(token, markdown)`。
- 使用 Milkdown `insert` / `replaceRange` 工具把 Markdown 插入到当前光标，而不是手动拼字符串。

建议新增 handle：

```ts
export type MarkdownNoteEditorHandle = {
	getMarkdown: () => string;
	focus: () => void;
	insertMarkdown: (markdown: string) => void;
	replaceMarkdown: (from: string, to: string) => void;
};
```

### 5.3 本地占位与替换

粘贴后立即生成临时 token：

```text
__uploading_asset_{assetLocalId}__
```

不同文件插入的临时 Markdown：

```md
![screenshot.png](blob:...)

<video controls src="blob:..."></video>

[report.pdf](blob:...)
```

上传成功后把 `blob:` 地址或 token 替换为返回的公开 URL：

```md
![screenshot.png](https://assets.notes.edmund.xin/...)

<video controls src="https://assets.notes.edmund.xin/..."></video>

[report.pdf](https://assets.notes.edmund.xin/...)
```

保存前需要检测正文中是否仍包含 `blob:` 或上传 token。如果存在：

- 正在上传：等待上传完成后再保存。
- 上传失败：提示“有附件上传失败，请重试或删除占位内容”，避免把本地 `blob:` 写入 D1。

### 5.4 新笔记静默持久化

当前 `saveDetailNote` 会保存后关闭详情页，不适合粘贴上传。新增 `ensureCurrentNoteForUpload`：

```ts
async function ensureCurrentNoteForUpload(markdownSnapshot: string): Promise<string>
```

逻辑：

- 如果已有 `currentNoteId`，直接返回。
- 如果没有，调用 `createCurrentNote` 创建笔记。
- 标题为空时使用 `未命名笔记`。
- 创建成功后更新 `currentNoteId`、`savedDetailTitle`、`savedDetailMarkdown`、notes 列表。
- 不关闭详情页。

### 5.5 嵌入入口 bridge

当前嵌入入口通过 iframe 与 AI 聊天页 bridge 做笔记 CRUD。需要扩展 action：

```ts
type NoteBridgeRequest =
	| { action: "list" }
	| { action: "create"; payload: { note: DraftNote } }
	| { action: "update"; payload: { id: string; note: DraftNote } }
	| { action: "delete"; payload: { id: string } }
	| { action: "uploadAsset"; payload: { id: string; file: File } };
```

主站同源时直接上传。跨源嵌入时由 iframe 内的 `ChatApp` 接收 `File` 并调用同源 API 上传。

## 6. 安全与限制

### 6.1 上传限制

v1 默认：

- 单文件最大 80 MiB。
- 单次粘贴最多 5 个文件。
- 禁止空文件。
- 禁止 `text/html`、`application/javascript`、`image/svg+xml` 等高风险类型。

允许类型：

- `image/png`
- `image/jpeg`
- `image/webp`
- `image/gif`
- `video/mp4`
- `video/webm`
- `audio/mpeg`
- `audio/wav`
- `application/pdf`
- `text/plain`
- `text/markdown`
- `application/zip`

后续 TODO：

- 大视频分片上传。
- 图片压缩和尺寸提取。
- 上传进度条。
- 附件管理面板。
- 未引用附件清理任务。

### 6.2 公开 URL 风险

选择公开 R2 URL 后，文件本身不再跟随笔记权限控制。降低风险的措施：

- R2 key 必须包含随机 `assetId`，不可使用可猜测路径。
- 不允许列 bucket 根目录。
- 生产使用自定义域名，不使用 `r2.dev`。
- 如果后续要做私密附件，需要切换到 Worker 私有代理或短期签名 URL。

## 7. Cloudflare 配置

`wrangler.jsonc` 增加：

```jsonc
{
	"vars": {
		"NOTE_ASSETS_PUBLIC_BASE_URL": "https://assets.notes.edmund.xin"
	},
	"r2_buckets": [
		{
			"binding": "NOTE_ASSETS",
			"bucket_name": "y-note-assets"
		}
	]
}
```

`EnvWithBindings` 增加：

```ts
type EnvWithBindings = Env & {
	wranglerdemo: D1Database;
	NOTE_ASSETS: R2Bucket;
	NOTE_ASSETS_PUBLIC_BASE_URL?: string;
	ASSETS?: Fetcher;
};
```

需要执行：

```bash
npx wrangler types
npx wrangler d1 migrations apply wranglerdemo
```

如果遇到：

```text
Please enable R2 through the Cloudflare Dashboard. [code: 10042]
```

说明账号还未启用 R2。此时上传功能不可用，但必须保证笔记 CRUD 不受影响。

## 8. 测试计划

### 8.1 Worker 测试

- 未登录上传返回 `401`。
- 上传到不存在的 note 返回 `404`。
- 上传到其他用户 note 返回 `404`。
- 禁止 MIME 返回 `400`。
- 超过大小返回 `413`。
- 成功上传后：
  - R2 存在对象。
  - D1 存在 `note_assets` 记录。
  - 返回 public URL 和 markdown。
  - `assetCount` 从 `0` 变为正确数量。
- 删除附件后：
  - D1 `deleted_at` 有值。
  - R2 对象被删除。
- 删除笔记后：
  - 笔记删除成功。
  - 关联 R2 对象被删除。

### 8.2 前端验证

- 粘贴普通文字不触发上传。
- 粘贴图片立即显示，上传成功后 URL 替换为公开域名。
- 粘贴视频生成 `<video controls>`。
- 粘贴 PDF 或 zip 生成附件链接。
- 新建未保存笔记时粘贴文件不弹窗、不关闭详情页。
- 上传失败时保留编辑内容并显示 toast。
- 保存时不会把 `blob:` URL 写入 D1。
- 主站和嵌入脚本入口行为一致。
